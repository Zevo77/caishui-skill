# 风控边界

做申报复核、最终提交准备或电子税务局实操时，读取本文件。

## 留痕要求

每个申报包应包含：

- 数据来源、文件名或哈希。
- 纳税人画像和关键假设。
- 政策证据卡。
- 计算底稿输出。
- 勾稽差异和处理说明。
- 用户对正式提交或缴款的确认。
- 用户完成提交后的回执、截图或申报状态。

## 人工确认节点

以下动作必须取得用户明确确认：

- 提交纳税申报。
- 缴款或发起银行扣款。
- 申请退税、抵税或留抵退税。
- 作废发票、开具红字发票、确认发票用途。
- 修改纳税人资格、发票额度、征收方式、银行账户等关键资料。

## 复核口径

用户要求 review/复核时，先列问题：

- 严重程度：阻断、高、中、低。
- 证据：文件、行号、字段、金额、来源、影响税种。
- 影响：可能少缴、多缴、申报退回、地方口径不确定或资料缺口。
- 修复：需要补充的数据、凭证、政策核验或用户确认。

## 禁止事项

不要提供：

- 保证节税结论。
- 偷逃税、虚开发票、虚列工资、虚列成本费用方案。
- 绕过电子税务局登录、验证码、CA/UKey、风控监测的步骤。
- 缺少官方来源和纳税人事实的确定性法律/税务结论。
