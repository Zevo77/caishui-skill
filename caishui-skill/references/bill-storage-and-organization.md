# 账单归档与整理

处理支付宝、微信支付、银行流水、发票、合同、工资社保、申报表、缴款回单、电子税务局回执等原始资料时，读取本文件。

## 原则

原始资料是申报、查账、税务稽核和内部复核的证据。不要把真实企业账单存放在 skill 目录。应按企业、年度、所属期、资料类型建立项目级归档，并保留原始文件、哈希、来源和整理记录。

不要删除、覆盖、改名原始文件。整理时只生成索引或复制归档副本。需要脱敏展示时，复制到临时目录后再脱敏。

## 推荐目录

```text
资料归档/
  2026/
    2026-06/
      01_发票/
      02_支付宝账单/
      03_微信账单/
      04_银行流水/
      05_合同订单/
      06_工资社保公积金/
      07_申报表和回执/
      08_完税缴款凭证/
      99_待分类/
    索引/
      账单索引.csv
      重复文件清单.csv
      缺资料清单.md
```

## 索引字段

账单索引至少包含：

- 文件ID：建议使用 SHA256 前 12 位。
- 原始路径、归档路径、文件名、扩展名、大小、修改时间。
- 推断日期、所属期、资料类别、来源平台、交易对方、金额、税额、发票号码、订单号。
- 关联凭证号、关联申报表、是否已入账、是否已勾稽、是否缺发票、备注。

`assets/templates/bill-index-template.csv` 提供基础表头。

## 整理步骤

1. 先扫描原始目录，计算哈希，生成索引。
2. 按文件名、路径和扩展名初步分类；分类不确定的标记为 `待分类`。
3. 检查重复哈希，保留原始来源，避免重复入账。
4. 把支付宝/微信/银行流水与发票、订单、账簿、回款进行勾稽。
5. 对缺发票、缺合同、缺回单、金额不一致、跨期、退款、手续费、拒付等事项生成待处理清单。
6. 月结或季报前冻结索引版本，作为申报底稿附件。

## 脚本

使用 `scripts/index_bills.py`：

```bash
python3 scripts/index_bills.py --source 原始资料目录 --out 账单索引.csv
python3 scripts/index_bills.py --source 原始资料目录 --out 账单索引.csv --copy-to 资料归档
```

脚本只扫描和复制，不删除原文件。复制归档时会按“类别/月份”建目录，并在文件名中加入日期、类别和哈希前缀，方便追溯。

## 风险提示

- 支付宝、微信、银行流水中的到账金额通常是净额，不能直接替代发票销售额或申报销售额。
- 平台手续费、退款、优惠、拒付、保证金、押金、代收代付要单独标记。
- 发票和流水不一致时，不要直接调收入；先查订单、退款、手续费、跨期和代收代付关系。
- 申报表、缴款凭证、电子税务局回执要长期留存，并能追溯到对应所属期和申报税种。
