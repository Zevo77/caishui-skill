# 纳税人数据

准备报表、计算税额或生成申报包前，读取本文件。

## 纳税人画像

要求或构造以下结构。优先使用中文键；脚本也兼容括号中的英文键。

```json
{
  "纳税人名称": "",
  "统一社会信用代码": "",
  "主体类型": "公司 | 个体工商户 | 合伙企业 | 分支机构 | 其他",
  "省份": "",
  "城市": "",
  "区县": "",
  "主管税务机关": "",
  "行业": "",
  "会计准则": "小企业会计准则 | 企业会计准则 | 其他",
  "增值税纳税人类型": "小规模纳税人 | 一般纳税人 | 未知",
  "征收方式": "查账征收 | 核定征收 | 定期定额 | 未知",
  "所属期起": "YYYY-MM-DD",
  "所属期止": "YYYY-MM-DD",
  "涉及税种": [],
  "发票模式": "数电票 | 增值税发票平台 | 混合 | 无 | 未知",
  "全国政策核验时间": "",
  "地方政策核验时间": ""
}
```

任何字段会影响税务结论时，必须询问、推断并标注假设，或把结论降级为“待确认”。

## 输入文件

优先使用结构化文件：

- 账簿 CSV：日期、科目编码、科目名称、借方、贷方、税务分类、发票号码、往来单位、摘要。
- 发票台账：开票日期、发票号码、发票类型、状态、销方/购方识别号、不含税金额、税额、价税合计、用途确认状态。
- 银行流水：交易日期、金额、对方户名、摘要、银行账号。
- 支付宝/微信/平台账单：交易时间、订单号、商户订单号、交易对方、收入金额、支出金额、手续费、退款、账单状态。
- 工资社保：员工、所属期、应税工资、社保、公积金、已扣个税。
- 历史申报和缴款回单：已申报销售额、增值税应纳税额、所得税应纳税所得额、附加税费、缴款凭证。
- 账单索引：文件ID、原始路径、归档路径、哈希、资料类别、所属期、是否已入账、是否已勾稽。

## 规范化规则

使用十进制金额计算。不要修改原始文件。把规范化结果另存，并保留原文件名、原行号和转换说明。

不要静默抵销异常差异。必须提示：

- 负收入、异常贷方余额、重复发票、作废/红冲发票、银行收款未匹配、工资个税缺失、关联交易、跨期截止问题。
- 发票金额、账面金额、银行流水、申报金额之间的差异。

## 隐私和凭据

不得索要或保存登录密码、UKey/CA PIN、短信验证码、人脸认证数据、银行扣款凭据或私钥。需要认证时，让用户在本机自行操作。
