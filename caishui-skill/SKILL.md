# 财税申报技能

## 核心规程

把中国大陆财税、申报、财务报表任务视为高风险任务。不要凭记忆回答税率、优惠政策、申报期限、电子税务局路径、发票规则或地方执行口径。凡涉及现代政策或平台操作，先查官方权威来源，再给结论。

严禁编造政策、文号、链接、发布日期、地方口径、申报路径或税务机关答复。不能核验时，输出“无法核验/待核验”，说明缺少什么来源或事实，并给出下一步核验方式。不要为了完整性补齐不存在的依据。

计算、申报或给出确定性政策判断前，先取得纳税人画像：主体类型、纳税人身份、所属省市区、主管税务机关、行业、所属期、涉及税种、增值税纳税人类型、征收方式、会计准则、发票模式和特殊资质。缺地区时，只能给全国口径，并标明“地方口径未核验”。

区分三类输出：

1. 工作底稿：表格、计算过程、勾稽关系和假设。
2. 申报包：电子税务局填报值、导入文件、申报前检查清单。
3. 官方动作：正式提交、缴款、退税、作废、红冲、用途确认等。没有用户明确授权和可追溯记录时，不得执行或引导不可逆动作。

所有税务结论都要给依据。列出来源标题、发布机关、文号、发布日期、施行/截止状态、适用地区、链接和核验日期。把“官方原文内容”和“基于事实作出的推断”分开写。

## 工作流

先判断任务类型：

- **政策问答**：读取 `references/policy-freshness.md` 和 `references/regional-source-strategy.md`。可联网时运行 `scripts/search_official_policy.py`。
- **账单/凭证整理**：读取 `references/bill-storage-and-organization.md`。需要建立资料索引或归档副本时，使用 `scripts/index_bills.py`。
- **财务报表**：读取 `references/taxpayer-data.md` 和 `references/statement-and-filing-workflow.md`。有账簿 CSV 时，可用 `scripts/build_financial_statements.py` 生成底稿。
- **申报准备**：读取 `references/statement-and-filing-workflow.md` 和 `references/controls-and-boundaries.md`。生成申报草稿和提交前检查清单，不做无人值守申报。
- **复核/风控**：读取 `references/controls-and-boundaries.md`。有结构化文件时，用 `scripts/validate_workpapers.py` 检查纳税人画像、报表平衡和增值税勾稽。

## 政策时效和地区口径

每次政策敏感回答都要：

- 按顺序核验：法律法规、财政部/国家税务总局公告、国家税务总局政策法规库、地方税务局公告、电子税务局办税指南、12366/官方问答。
- 优先匹配“所属期适用的规则”，而不是只看最新页面。
- 检查 `自...起施行`、`执行至`、`废止`、`失效`、`部分条款废止`、`另有规定`、`由省级税务机关确定` 等关键词。
- 最低证据标准：至少有一个可打开的官方来源；涉及地方差异时，还要有主管地区或上级地方税务机关来源。只有新闻稿、培训材料、第三方文章或模型记忆时，不得给确定性结论。
- 全国口径和地方材料冲突，或地方材料缺失时，明确说明冲突/缺口，并建议向主管税务机关确认。

## 数据和安全

不要索要、保存或转述密码、CA/UKey PIN、短信验证码、人脸认证信息、银行扣款凭据、私钥。涉及电子税务局实操时，默认只做导航辅助、截图核对、草稿准备或导入文件生成；除非用户给出明确授权，否则不得提交或缴款。

使用 `assets/templates/` 中的模板作为纳税人画像、账单索引、账簿、发票台账、增值税勾稽和申报复核清单的起点。脚本支持关键模板的中文表头。真实账单和凭证不要存进 skill 安装目录，应存放在当前企业/项目的资料归档目录中。

## 输出格式

实质性输出按以下顺序组织：

1. 范围：纳税人、地区、所属期、税种、是否已核验地方口径。
2. 结果：简明答案或生成文件/表格摘要。
3. 依据：官方证据卡和关键假设。
4. 检查：勾稽差异、缺失数据、地方不确定性、阻断申报的风险。
5. 下一步：正式申报前还需要的数据、核验或用户确认。
